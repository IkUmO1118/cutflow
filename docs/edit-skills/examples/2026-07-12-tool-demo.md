## この記録の読み方

この収録は **`reasonId` フィールド導入前**に編集されたもの(2026-07-12。
`reasonId` は本プログラムの P2 で追加された)。したがって `cutplan.json` の
実データに `reasonId` は1件も存在しない。ここに載せる各判断 JSON の
`reasonId` は、実際の `reason`(文字列。`cutplan.json` の値そのまま)に対して
**この記録を書く際に後付けで分類した分析**である——`reasonId` の値そのものが
生成当時のデータであるかのように読まないこと。`id` はこの記録内だけで通用する
通し番号(元の LLM 呼び出しの候補番号ではない。生成当時の `plan.raw.txt` は
以後の再実行で上書きされ残っていない)。`reason` に残る `#NN` という数字
だけが当時の実際の値。

`start`/`end` は**元収録の秒**(CLAUDE.md の時刻規約どおり)。`text` フェンスの
発話は `transcript.json` から改変せずそのまま引く。
