# greeting

> 系: 残す · 既定の処置: keep
> 接地: 実データ(反転keep, 2026-07-12) + rules.md

## 一行定義

挨拶・お礼・締め。**尺の都合で切らない**。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 「ご視聴ありがとうございました」「よろしくお願いします」
  「それでは」等の定型の挨拶・締めの言い回し
- **補助**: 前の話題(本編の結論)から独立した、視聴者への呼びかけの体裁を
  取っている

### 時間・格子(候補の形)

- **補助**: 動画の先頭または末尾に位置する。特に末尾(最後の keep 候補
  付近)にあるものは高確率で `greeting`

### 音(plan.perception.audio)

- 決定的シグナルではない。ただし挨拶の直後に残る間(息の切れ目・余韻)は
  `dead-air` と誤判定されやすい(下記反例)

### 画面(plan.perception.ocr / frames)

- 決定的シグナルではない

## 既定の処置

`keep`。テンポ調整・尺の都合を理由に切らない。

## 反例(この分類を当てない場合)

- **本編中の「それでは」「では」等**。文頭の接続として使われているだけで
  締めの文脈でないなら `greeting` ではない(語彙が同じでも位置と文脈で
  判断する)
- **実際に起きた失敗(`rules.md`、2026-07-17)**: AI が締めの挨拶を丸ごと
  カットした。`rules.md` はこれを受けて「挨拶は必ず残す。締めの『ご視聴
  ありがとうございました』等のお礼・挨拶の発話はテンポ調整の対象にしない」
  と明記している
  → `~/Movies/cutflow/2026-07-12` の最終 `cutplan.json` には、この失敗の
  修正と思われる痕跡が残る。締めの発話「それでは」(`[576.84-578.11]`)の
  直後、`[578.11-580.45]` の2.34秒が `action: "keep"` かつ
  `reason: "無音"`(AI が cut と判断した理由がそのまま残った反転)として
  残っている。この収録は「それでは」を最後に録画が終わっており
  (`manifest.json` の `durationSec` は 582.033)、直後の無音は挨拶の
  余韻——録画を止める直前の間——であり、`dead-air` として詰めると
  締めの一言が唐突に切れて終わる印象になる

## 紛らわしい隣

- `hook` — 動画の反対の端(冒頭)にある、同じく「テンポ調整の対象に
  しない」区間
- `dead-air` — 挨拶の直後に残る間は無言という表層だけ見ると `dead-air`
  に見えるが、締めの発話に付随する余韻は `greeting` の一部として残す
  (上記反例)
- `slate` — 締め挨拶の発話そのものは `greeting`、その発話が終わった
  **後**(録画終了までの完全な無音)は `slate`。この収録では
  `[578.11-580.45]`(挨拶の余韻。keep)と `[580.45-582.03]`(録画終了までの
  残り。cut)が隣接しており、境界の引き方が両分類の違いを示す好例になる

## worked example

`~/Movies/cutflow/2026-07-12` の締め。`rules.md` の「末尾は『おわりに』で
静かに締める」の型どおり。

```text
[573.82-577.94] "また進捗があったら続きも取ろうと思います"
[576.84-578.11] (keep) "それでは"
[578.11-580.45] (keep, reason="無音") ※挨拶の余韻。2.34秒
[580.45-582.03] (cut, reason="無音"/"") ※録画終了までの残り(slateの対象)
```

`[576.84-578.11]` の「それでは」自体はもちろん残すが、その直後の無音
`[578.11-580.45]` も**挨拶の一部として残った**点が `greeting` の実例として
重要——尺(2.34秒)だけ見れば `dead-air` の候補になりうる長さだが、
締めの発話に直接続く間は切らない判断がされている。

```json
{ "id": 91, "reasonId": "greeting", "reason": "締めの挨拶。尺の都合で切らない" }
```
