# dead-air

> 系: 切る · 既定の処置: cut
> 接地: 実データ(2026-07-12, 2件) + rules.md

## 一行定義

本編内で、**発話も画面の新情報もない**長めの間(段取り・思考・移動)。
母艦の旧 `setup-silence` を統合済み(理由は [README](../README.md) の G1)。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 候補に発話が無い(無音)
- この軸だけでは `demo-wait` と区別できない。**必ず画面の軸と併用する**

### 時間・格子(候補の形)

- **補助**: 尺 > 1.5秒(`gap-trim` との弁別。ただし短くても前後どちらかが
  cut/存在しない=発話の継ぎ目でないなら `dead-air` を優先してよい)
- 動画中盤に多い(段取り・考え込みの間は冒頭・末尾より本編中に出やすい)

### 音(plan.perception.audio)

- **補助**: `plan.perception.audio: true` のとき無音区間として注記される。
  `gap-trim` と同じ注記を受けるため、単独では使わない

### 画面(plan.perception.ocr / frames)

- **決定的**: 画面 OCR / frames が**直前候補から実質変化していない**
  (=話者が何も操作していない・見せていない)
- **この決定的シグナルが取れないとき(既定オフ環境・OCR 非対応環境)は
  `demo-wait` を疑う側へ倒す。** 画面が語っているかどうかを確認できない
  状況で無言を切ると、`rules.md` が記録する最大の失敗(下記反例)を
  再現するリスクがある

## 既定の処置

`cut`。

## 反例(この分類を当てない場合)

- **`rules.md` に記録済みの実際の失敗(2026-07-17)**: 「AI がデモ中の無言
  画面を『静止画面の待ち時間』として大量カットした」。GUI エディターの
  デモ操作中、画面には結果が表示されているのに `dead-air` と誤判定して
  切ってしまった。正しくは `demo-wait`(画面が語っている無言)。
  → この失敗は `~/Movies/cutflow/2026-07-12` の最終 `cutplan.json` にも
  痕跡が残る。`action: "keep"` なのに `reason: "無音"` のまま残っている
  8箇所のうち4箇所(`[448.29-448.59]` `[457.78-458.30]` `[465.46-465.91]`
  `[468.45-468.87]`。詳細は `demo-wait` の worked example)は、AI が
  `dead-air` として cut 提案したものを人間が GUI operator として keep へ
  戻した反転の記録である
- **発話が無くても、画面が直前から変わっている場合**は `dead-air` を
  当てない(`demo-wait` を検討する)
- **候補の前後が両方 keep で両方発話を持つ短い無音**は `dead-air` では
  なく `gap-trim`(テンポ調整の間)

## 紛らわしい隣

- `demo-wait` — 唯一の弁別子は**画面の変化の有無**。表層(無言)は同じだが
  処置は正反対(G2)。判断に迷ったら残す側(`demo-wait`)に倒す
  (`rules.md` の失敗記録がその根拠)
- `gap-trim` — 弁別子は尺と隣接候補。短く(1.5秒以下)、前後とも発話中の
  候補に挟まれるなら `gap-trim`
- `greeting` — 締めの挨拶の直後に残る無言は `dead-air` に見えるが、挨拶の
  余韻として残す判断がありうる(`greeting` の worked example を参照)
- `slate` — 本編内の間か、収録の頭/尻(本編の外)かで区別する。`slate` は
  位置(先頭区間・最終 keep より後)が決定的

## worked example

ことはシンプルで」の間、約21.3秒の間(考えをまとめている・次の説明の

```text
[39.74-40.42] (cut, 0.68s, reason="発言なしの間")
[40.42-45.72] (cut, 5.30s, reason="無音")
[45.72-46.46] (cut, 0.74s, reason="発言なしの間")
[46.46-54.51] (cut, 8.05s, reason="無音")

```

この間、画面は動いていない(話者が次の説明を切り出す前の思考の間で、

```json
{ "id": 9, "reasonId": "dead-air", "reason": "発言なしの間" }
```
