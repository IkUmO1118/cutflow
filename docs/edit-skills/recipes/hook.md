# hook

> 系: 残す · 既定の処置: keep
> 接地: 実データ(2026-07-12) + rules.md

## 一行定義

冒頭のフック(課題提示・つかみ)。テンポ調整の対象にしない。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 動画先頭付近で、**主題(ツール名・具体的な解決策)を提示する
  前**の課題提起・共感の発話(「〜って地味に大変ですよね」「〜って結構
  理不尽じゃないですか」のような、視聴者の共感を誘う言い回し)
- **補助**: 「というわけで」「そこで」等、課題提起から主題への転換を示す
  遷移句が続く場合、その直前までが `hook`

### 時間・格子(候補の形)

- **決定的**: 動画の**先頭 ~20秒**(`rules.md` の型)、かつ主題提示より前。
  位置(先頭)と内容(主題未提示)の両方を満たすことが条件——先頭でも
  既に主題が出ていれば `hook` の範囲外

### 音(plan.perception.audio)

- 決定的シグナルではない

### 画面(plan.perception.ocr / frames)

- 決定的シグナルではない。`hook` はほぼ発話のみで成立する判断

## 既定の処置

`keep`。テンポ調整(`gap-trim` 等)で間を詰めることはあっても、`hook` を
理由に候補全体を切ることはしない。

## 反例(この分類を当てない場合)

- **先頭付近でも、既に主題(ツール名・具体的な解決策)が出ている場合**は
  `hook` ではなく通常の keep(本編)。`hook` は「主題提示より前」という
  条件が必須
- **20秒を大きく超えて課題提起が続く場合**。冗長な前置きは `hook` の
  保護対象ではなく、通常のカット判断(`restatement` / `tangent` 等)の
  対象になりうる

## 紛らわしい隣

- `greeting` — 動画の反対の端(末尾)にある、同じく「テンポ調整の対象に
  しない」区間。`hook` は冒頭の課題提起、`greeting` は末尾の挨拶で、
  位置で区別する

## worked example

`~/Movies/cutflow/2026-07-12` の冒頭。`rules.md` の「冒頭フックは約20秒
(課題への共感で始め、その後に主題を出す)」の型どおり、共感 → 主題提示の
順で構成されている。

```text
[3.60-6.52]   "動画編集って地味に時間かかりますよね"
[6.95-10.24]  "しかも最近はAIにやらせようとなると"
[10.76-13.93] "今度はAPI代が地味に効いてくる"
[13.93-19.29] "時間もお金も両方持っていかれてるって結構理不尽じゃないですか"
[20.31-23.48] "というわけでクロードコードを使って"  ← ここから主題提示
[24.01-30.70] "時間もお金もあんまりかけずに動画編集を自動化する仕組みを作りました"
```

`hook` の範囲は `[3.60-19.29]`(課題提起・共感)。`[20.31-]` の「という
わけで」以降は主題提示であり、`hook` の判定対象ではなくなる(通常の
本編として扱う)。冒頭の無音・言い直し断片(`[0-3.6]` の `slate`、
`[6.52-8.81]` 付近の重複断片)は `hook` 本体ではなく別分類。

```json
{ "id": 2, "reasonId": "hook", "reason": "冒頭の課題提起。主題提示より前" }
```
