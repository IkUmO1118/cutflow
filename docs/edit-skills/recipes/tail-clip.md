# tail-clip

> 系: 境界 · 既定の処置: boundary(後ろへ)
> 接地: rules.md + 実データ(margin, 2026-07-12)

## 一行定義

カット境界が**語尾を食う**。切らずに余白を残す側へ倒す。`boundary` 系
(「切るか残すか」ではなく「境界をどちらへずらすか」)。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 候補の先頭(=直前 keep 候補の直後に来る cut 候補の開始点)が、
  直前発話の語尾(「です」「ます」「ました」等の助動詞・活用語尾)の
  **途中**に食い込んでいる
- **補助**: 直前 keep の末尾テキストが助動詞・活用語尾の途中で不自然に
  終わっている(文字起こし上、語が完結していない)

### 時間・格子(候補の形)

- **決定的**: 境界(keep の `end`)が、その直前の語の**発話終了時刻より
  前**にある(=語の途中で切れる)。境界は語の終了時刻**以降**、かつ
  0に近すぎない余白を残した位置にあるべき
- **補助**: `rules.md` の目安は「keep 内に残す間は最小限(無音は出力尺の
  3%程度まで)」——余白はゼロにする必要は無いが、大きく空ける必要も無い

### 音(plan.perception.audio)

- 決定的シグナルではない。境界の攻めすぎは語彙・時間の軸で判定する

### 画面(plan.perception.ocr / frames)

- 決定的シグナルではない

## 既定の処置

`boundary`(後ろへ)。境界を切らずに、語尾が言い切られた後まで残す側へ
ずらす。

## 反例(この分類を当てない場合)

- **境界が既に語の終了時刻より後にあり、かつ極端に長い余白でもない場合**は
  `tail-clip` の対象ではない(正常な境界)
- **`rules.md` に記録された一般的な戒律(日付・元秒の記録なし)**:
  「カット境界は攻めすぎない。語尾を絶対に食わないこと(『〜です』の
  『す』が消えるのが過去の失敗で最多)。発話の終端ぴったりではなく、語尾が
  言い切られた後に少し余白を残してから切る。境界で迷ったら『残す側』に
  倒す」。この失敗は `demo-wait` / `greeting` の反例(2026-07-17 の反転
  keep)と違い、**特定の日付・特定の元秒を伴うインシデントとして
  `rules.md` に記録されていない**(「過去の失敗で最多」という累積的な
  傾向として記されている)。よって本 recipe の反例節は、`rules.md` の
  一般則を**この収録の実測データ**(下記 worked example)に当てはめて
  具体化したものであり、`demo-wait` / `greeting` のような「AI の誤判定が
  人間に反転された」個別インシデントの引用ではない

## 紛らわしい隣

- `reference-orphan` — 境界系のもう一方。`tail-clip` は境界を**後ろへ**
  (残す方向)、`reference-orphan` は境界を**前へ**(参照を守る方向)ずらす
- `restatement` — `restatement` で前半を切るときも、切る境界が収束先の
  直前の語尾に食い込んでいないか確認する必要がある(`restatement` の
  「紛らわしい隣」を参照)

## worked example

`~/Movies/cutflow/2026-07-12` の `cutplan.json`。文が言い切られた直後に
残る余白の実例。

```text
[366.44-373.48] whisper上の発話区間 "こういう画面がブラウザで開きます"
[371.20-371.40][371.40-373.25] (keep, reason="#50末尾の重複") 発話の末尾部分
[373.25-373.72] (keep, reason="無音") ※文が言い切られた後の余白。0.47秒
[373.72-377.96] (cut, reason="無音")
```

「開きます」を言い切った直後の `[373.25-373.72]` は、AI が一度 cut と
判断した(`reason` に `"無音"` が残っている)にもかかわらず `action: "keep"`
のまま残っている。もしこの余白を削って境界を `373.25` 付近(語の完了
直後)まで詰めていたら、`rules.md` が警告する「語尾を食う」リスクに
近づく——実際にどの程度で「食う」に転じるかは音素の継続時間に依存するため
一律の秒数では線引きできないが、**余白をゼロに詰めない**という判断が
この収録全体で一貫している(keep の平均尺 1.84秒・114件、無音は出力尺の
3%程度に抑えられている、という §0.4 の実測と整合する)。

```json
{ "id": 51, "reasonId": "tail-clip", "reason": "語尾の後に余白を残して境界を後ろへ" }
```
