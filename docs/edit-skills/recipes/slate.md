# slate

> 系: 切る · 既定の処置: cut
> 接地: 実データ(2026-07-12, 2件)

## 一行定義

収録の頭/尻の段取り。**本編の外**(カメラ操作・「じゃあ撮ります」・締め
挨拶より後の残り)。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 発話が無い、または撮影メタ発話(カメラ操作・録画開始/終了の
  合図)のみ
- `dead-air` と語彙面では区別できないことが多い。**位置**が唯一の決定的
  弁別子(下記)

### 時間・格子(候補の形)

- **決定的**: 動画の**先頭区間**(最初の keep 候補より前)、または
  **最終 keep 候補より後**(締め挨拶等、最後に残る発話より後ろ)
- この位置条件が `dead-air`(本編内)との唯一の弁別子。同じ「無言」でも
  位置が本編の内か外かで分類が変わる

### 音(plan.perception.audio)

- 決定的シグナルではない。`dead-air` と同じ注記を受ける

### 画面(plan.perception.ocr / frames)

- 通常は無関係(録画開始/終了直後は静止していることが多い)

## 既定の処置

`cut`。

## 反例(この分類を当てない場合)

- **先頭区間だが、既にフックの発話が始まっている場合**は `slate` ではなく
  `hook`。位置条件(先頭区間)だけで判定せず、発話の有無を必ず確認する
- **最終 keep より後ろだが、締め挨拶自体がその区間に含まれる場合**は
  `slate` ではなく `greeting`。締め挨拶の発話そのものは切らない
  (`rules.md` の失敗記録は `greeting` の worked example を参照)

## 紛らわしい隣

- `dead-air` — 弁別子は位置。本編内(最初の keep 〜最後の keep の間)なら
  `dead-air`、本編の外(先頭区間・最終 keep より後)なら `slate`
- `greeting` — 動画の終端付近という点で近いが、`greeting` は**発話その
  もの**(挨拶の言葉)、`slate` はその**発話が終わった後の余剰**を指す

## worked example

`~/Movies/cutflow/2026-07-12` の頭と尻。

頭(録画開始〜最初の発話まで。発話ゼロの完全な無音):

```text
[0.00-3.60] (cut, reason="無音") — 最初の発話「動画編集って地味に時間かかりますよね」は 3.60 開始
```

尻(締めの挨拶「それでは」の後、収録終了まで。動画全体の尺は 582.033秒
= manifest.json の `durationSec`):

```text
[576.84-578.11] (keep) "それでは"
[578.11-580.45] (keep, reason="無音") ※greeting の余韻。詳細は greeting の worked example
[580.45-581.37] (cut, reason="無音")
[581.37-582.03] (cut, reason="") ※収録ファイルの末尾。この直後にファイルが終わる
```

`[580.45-582.03]` は締め挨拶「それでは」の発話そのものではなく、その後に
残った録画終了までの無音(カメラを止める・収録アプリを操作している間)で、
最終 keep([578.11-580.45] の末尾)より後ろに位置する。

```json
{ "id": 79, "reasonId": "slate", "reason": "締め挨拶後、収録終了までの残り" }
```
