# stumble

> 系: 切る · 既定の処置: cut
> 接地: 実データ(言い淀み系に同居。2026-07-12) + craft

## 一行定義

内容を1つも運んでいないフィラー単独(えー/あの/まあ/なんか)、または
同じ句を言い淀んで繰り返すだけの候補。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 候補テキストがフィラー語のみ、または**直前候補と同一・ほぼ
  同一の句の繰り返し**で、除いても前後の文が成立する(=候補を消しても
  情報が失われない)
- **補助**: 「えー」「あの」「まあ」「なんか」等の典型フィラー語。繰り返し型
  では句の先頭〜途中で切れて再開している(「保存すればその場でホット
  リロード」→「その場で反映されます」のように、言い切れず再挑戦している)
- **補助(craft)**: フィラー除去は編集の最も基本的な技法の一つだが、
  弁別の基準は語彙リストの一致ではなく**「除いても意味が変わらないか」
  だけ**、という一般則がある。フィラーに似た語でも内容を運んでいれば
  残す(下記反例)。`restatement` との弁別も同じ基準(除去後に情報が
  失われないなら `stumble`、失われるなら `restatement`)で説明できる

### 時間・格子(候補の形)

- **補助**: 尺 < 1.0s(フィラー単独の典型)。繰り返し型は複数の短い候補が
  無音を挟んで連続する(下記 worked example のように4候補が並ぶ)
- 決定的シグナルではない。尺だけでは `gap-trim`(無音区間)と区別できない
  ——`stumble` は候補内に**発話そのもの**がある点が違う

### 音(plan.perception.audio)

- **補助**: 繰り返し型では候補間に短い無音(息継ぎ・言い淀みの間)が挟まる。
  `plan.perception.audio: true` のとき注記される
- 代替(既定オフ時): 語彙シグナル(内容の欠如・繰り返し)だけで判定できる

### 画面(plan.perception.ocr / frames)

- 通常は無関係。画面が同時に変化している場合は `demo-wait` を先に検討する

## 既定の処置

`cut`。

## 反例(この分類を当てない場合)

- **繰り返しに見えて、後の版が前の版の言い換え・拡張になっている場合**。
  単なる同一句の反復ではなく、内容が発展・収束しているなら `stumble` では
  なく `restatement`(収束前の版)。弁別子は「除いても情報が失われないか」
  ——`stumble` は消しても何も失われないが、`restatement` の前半は
  「言い直しの過程」として意味を持つことがある
- **フィラーに見える語が実は本題の一部の場合**。「なんか変な感じがする」の
  ように「なんか」が発話の内容(違和感の表現)を担っているなら、フィラー
  単独ではないので `stumble` を当てない
- この収録には「えー」「あの」のような**教科書的な単独フィラー語**が候補
  として独立に切られた例は無い(whisper の書き起こしはフィラー音を明示的な
  トークンとして残さない傾向があるため)。実データの接地は下記 worked
  example の**同一句の繰り返し**(言い淀みの一種)に限られる

## 紛らわしい隣

- `restatement` — 弁別子は内容の有無。`stumble` は繰り返すだけで内容が
  進まない。`restatement` は後続候補が本候補の内容を包含・言い換える
  (内容が収束する)

## worked example

`~/Movies/cutflow/2026-07-12` の「GUI エディターの利点」を説明する場面。
話者は同じ言い回しを4回言い淀んでから、最終的に別の言い回しへ切り替えて
言い切っている(この最終版は本 worked example の範囲外。`restatement` の
worked example にある `[433.29-...]` 以降の一息版がそれにあたる)。

```text
[414.79-417.43] "らカットの位置やテロップの"
[418.96-421.09] "いち調節するみたいなことは要りません"
[425.19-427.73] "ばその場でホットリロードその"
[428.17-429.08] "で反映されます"
```

いずれも `reason: "同じ句の繰り返し(言い淀み)"`(実際の `cutplan.json` の
値)。除いても前後の文脈(直前の `JSONを直接見ながら...` と、直後の
`JSONを直接見ながらカットを行うのではなく...`)は成立する。

```json
{ "id": 58, "reasonId": "stumble", "reason": "同じ句の繰り返し(言い淀み)" }
```
