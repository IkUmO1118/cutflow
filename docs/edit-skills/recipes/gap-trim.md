# gap-trim

> 系: 切る · 既定の処置: cut
> 接地: 実データ(2026-07-12, 35/38件) + craft

## 一行定義

keep された発話と発話のあいだの**短い無音を詰める**(テンポ調整)。
言い直しでも脱線でもなく、単に呼吸・継ぎ目の間を短くする処置。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 候補が発話をほとんど含まない(無音、または息継ぎ音程度)
- **決定的**: **前後の隣接候補が両方とも keep で、両方とも発話を持つ**
  (=候補の前後に切れ目なく会話が続いている)。これが `dead-air` との
  唯一の弁別子

### 時間・格子(候補の形)

- **決定的(補助扱いだが強い実測根拠)**: 尺 0.1〜1.5秒。実データでは
  無ラベル38件のうち35件(92%)がこの帯に入る(尺分布: `<0.5s` 23件 /
  `0.5–1.5s` 12件 / `>1.5s` 3件)
- keep の平均尺(1.84秒・114件)と比べて明確に短い
- **補助(craft)**: 「呼吸の単位」という考え方——発話と発話の継ぎ目の無音を
  ゼロまで詰めると、不自然な連続音(息継ぎの無い機械的な喋りに聞こえる)
  になる、という編集の一般則がある。詰めた後も**最小限の呼吸分は残す**。
  この craft は帯(0.1〜1.5秒)の**理由付け**であって、決定的シグナルの
  実測帯そのものを変えるものではない(§5.3。実データが優先)

### 音(plan.perception.audio)

- **補助**: `plan.perception.audio: true` のとき、候補が無音区間として
  注記される。ただし `dead-air` も同じ注記を受けるため、音だけでは
  `dead-air` と区別できない(語彙・時間・格子の決定的シグナルと併用する)
- 代替(既定オフ時): 前後候補が両方 keep で両方発話を持つ、という語彙側の
  決定的シグナルだけで判定できる

### 画面(plan.perception.ocr / frames)

- 通常は無関係(短い継ぎ目の間に画面が意味のある変化をすることは稀)。
  仮に画面変化があっても、尺が短い(< 1.5s)場合は `gap-trim` を優先してよい

## 既定の処置

`cut`。

## 反例(この分類を当てない場合)

- **前後どちらかが cut の場合**。片方が発話を持たない/存在しないなら
  「発話と発話の継ぎ目」ではないので `gap-trim` ではなく `dead-air` を
  検討する(決定的シグナルの前提条件が崩れる)
- **尺が 1.5秒を大きく超える場合**。実データでも `>1.5s` は3/38件のみで
  少数派。長い間は「テンポ調整」の範囲を超え、段取りや思考の間である
  可能性が高いので `dead-air` を優先する
- 実データでは無ラベル38件のうち8件は前後どちらとも接しない完全な無音
  (発話ゼロ)だった。これらは `gap-trim` ではなく通常の `無音` カット
  (どの分類にも当たらない、単なる録音の空白)として扱う

## 紛らわしい隣

- `dead-air` — 弁別子は**尺**と**隣接候補の発話有無**。`gap-trim` は
  短く(概ね1.5秒以下)、前後とも発話中の候補に挟まれる。`dead-air` は
  長く、発話も画面変化も無い

## worked example

`~/Movies/cutflow/2026-07-12` の2箇所。いずれも `cutplan.json` の `reason`
は空文字(=人間の GUI 操作による無ラベルカット。§0.2 の新事実)。

例1(0.53秒。冒頭の主張の切り替わり):

```text
[22.02-23.48] (keep) "というわけでクロードコードを使って"
[23.48-24.01] (cut, 0.53s, reason="")
[24.01-26.31] (keep) "時間もお金もあんまりかけずに動画編集を自動化する仕組みを作りました"
```

例2(1.39秒。2つの raw segment に分裂):

```text
[433.29-434.67] (keep) "JSONを直接見ながら"
[434.67-434.81] (cut, 0.14s, reason="")
[434.81-436.06] (cut, 1.25s, reason="無音")
[436.06-437.36] (keep) "カットを行うのではなく"
```
