# tangent

> 系: 切る · 既定の処置: cut
> 接地: 想定 · 観測: 2収録とも0件

**この分類は `~/Movies/cutflow/2026-07-12` に実例が無い(想定由来)。**
承認済み収録が現状1本しかなく(母艦 §6.1)、その1本は「ツール紹介・デモ」
という脱線の少ない構成だったため、接地が取れていない。以下の判定シグナルと

## 一行定義

本題と無関係な脱線・独り言・視聴者に向いていない発話。

## 判定シグナル

### 語彙(transcript)

- **決定的**: `brief.md`(今回の見せ場・中身)や `chapters.json` の章立てが
  示す主題のどれにも紐づかない話題への逸脱
- **補助**: 一人称の独語(「あれ」「どうだったっけ」「あ、そうそう」)。
  視聴者への語りかけではなく自分に向けた発話

### 時間・格子(候補の形)

- **補助**: 話題が戻ってくる(脱線の前後で同じ話に復帰する)場合、脱線
  区間の境界が明確になりやすい。話題が戻らないまま動画が終わる場合は
  `tangent` ではなく構成そのものの問題(分類の対象外)

### 音(plan.perception.audio)

- 決定的シグナルではない。脱線かどうかは無音の有無では判定できない

### 画面(plan.perception.ocr / frames)

- **補助**: 画面操作が本題と無関係な操作(SNS の通知確認・関係ないウィンドウ
  切り替え等)を映している場合、脱線の裏付けになる。ただし画面が本題の
  デモを継続している場合は、発話だけが脱線していても慎重に判断する
  (デモの補足説明を脱線と誤判定しない)

## 既定の処置

`cut`。

## 反例(この分類を当てない場合)

- **一見脱線に見えて、実は本題への布石(フック)の場合**。冒頭で「そういえば
  最近こんなことがあって」と個人的な話から始まり、そこから課題提起に
  つながる構成は `tangent` ではなく `hook` の一部
- **失敗の話が脱線に見える場合**。エラーに遭遇して「あれ、なんでだろう」と
  試行錯誤する発話は、内容が本題(ツールの使い方)に直結しているなら
  `tangent` ではなく `failure-and-fix`(残す)を検討する

## 紛らわしい隣

- `failure-and-fix` — エラー対応中の独語・試行錯誤の発話は脱線に見えるが、
  本題に直結しているので `failure-and-fix` を優先する
- `restatement` — 話題を変えて戻ってくる動きが、単なる言い直しの反復に
  見えることがある。`tangent` は**別の話題**へ逸れる、`restatement` は
  **同じ内容**を繰り返す・言い直す

## worked example

**接地なし。合成例。** 「JSON でカット編集する仕組み」を説明している最中に、
無関係な近況(ランチの話)へ逸れてから本題に戻る、という想定シナリオ。

```text
[想定 120.0-126.5] "そういえば今日のお昼、駅前に新しくできたラーメン屋
行ったんですけど、めちゃくちゃ並んでて結局諦めちゃって"
[想定 126.5-127.0] (無音)
[想定 127.0-131.2] "えっと、それで、JSONの話に戻ると"
```

最初の候補([120.0-126.5])は章立て・brief のどの見せ場にも紐づかず、
直後で話者自身が「それで、JSONの話に戻ると」と本題への復帰を明言している
(=脱線であったことが発話自体から確認できる)。

```json
{ "id": 31, "reasonId": "tangent", "reason": "本題と無関係な近況の脱線" }
```
