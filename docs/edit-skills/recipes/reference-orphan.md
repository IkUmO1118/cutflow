# reference-orphan

> 系: 境界 · 既定の処置: boundary(前へ)
> 接地: 想定 + rules.md(ズーム節の指示語列)

**この分類の判断そのもの(候補を切った結果、指示語が宙に浮いた実例)は
`~/Movies/cutflow/2026-07-12` の `cutplan.json` に無い(想定由来)。**
ただし指示語で画面の具体物を指す発話パターン自体は `rules.md` のズーム節に
実例がある(下記)。判定シグナルと worked example はこのパターンと一般的な
編集知識から構成した**合成例**である(G4)。

## 一行定義

ここを切ると**後続の指示語・接続が宙に浮く**。`boundary` 系
(「切るか残すか」ではなく「境界をどちらへずらすか」)。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 後続 keep 候補の先頭に**指示語**(「これ」「それ」「この
  JSON」「そっち」)や**接続詞**(「なので」「だから」)があり、その
  指示対象・論理的な前提が**本候補にしかない**
- **補助**: `rules.md` のズーム節が実例として挙げる指示語パターン
  (「この JSON を」「このエディターで」「このチェックボックスを」
  「フォルダの中身は」)——これらは話者が画面の具体物を指す瞬間の定型で、
  直前に「何を指しているか」を説明する発話が無いと、視聴者は指示語の
  対象が分からなくなる

### 時間・格子(候補の形)

- **決定的**: `reference-orphan` は**単一候補では判断できず、次の keep
  候補とセットで見る**(G3: 本候補を切ってよいかは、次の候補の先頭を
  確認しないと分からない)

### 音(plan.perception.audio)

- 決定的シグナルではない

### 画面(plan.perception.ocr / frames)

- **補助**: 画面 OCR / frames で、指示語が指す対象(ズームで示される
  具体物・強調されているファイル名等)が確認できる場合、参照関係の裏付けに
  なる

## 既定の処置

`boundary`(前へ)。境界を切らずに前へずらし、指示対象を説明する発話を
残す。

## 反例(この分類を当てない場合)

- **指示語があっても、その対象が既に別の keep 候補で説明済みの場合**は
  `reference-orphan` を当てない(参照は既に成立している)
- **指示語がフィラー的な用法の場合**(「それで」が単なる話の継ぎ穂で、
  具体的な対象を指していない)。この場合は指示語があっても参照は宙に
  浮かないので `reference-orphan` の対象にしない

## 紛らわしい隣

- `tail-clip` — 境界系のもう一方。`tail-clip` は境界を**後ろへ**(語尾を
  守る方向)、`reference-orphan` は境界を**前へ**(次候補の参照を守る
  方向)ずらす
- `duplicate-tail` — 末尾の重複断片を切った結果、直後の候補の指示語の
  参照先が本候補にしかない場合は `reference-orphan` の判定を優先する
  (`duplicate-tail` の「紛らわしい隣」を参照)

## worked example

**接地なし(合成例)。** `rules.md` のズーム節の実例パターンに基づく想定
シナリオ。ズームは「話者が指示語で画面の具体物に言及・操作している瞬間
だけ」入れる、という `rules.md` の記述と対になる——指示語の対象を説明する
発話を切ってしまうと、ズームで見せている対象と発話が噛み合わなくなる。

```text
[想定 250.0-253.8] "バリデートを走らせると、このJSONのここが直る前に
エラーで止まります"
[想定 253.8-254.2] (無音)
[想定 254.2-257.0] "このチェックボックスをオンにすれば自動修正されます"
```

`[254.2-257.0]` の「このチェックボックスを」の「この」は `[250.0-253.8]`
で説明された対象(JSON のエラー箇所を示す UI)を指す。`[250.0-253.8]` を
無音・冗長と誤判定して切ると、「このチェックボックス」が何を指すか視聴者に
伝わらなくなる。

```json
{ "id": 55, "reasonId": "reference-orphan", "reason": "次候補の指示語『この』の参照先" }
```
