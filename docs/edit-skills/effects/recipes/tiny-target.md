# tiny-target

> 系: 見せる · 既定の型: zoom
> 接地: 実データ(2026-07-12, 4件) + rules.md + craft

## 一行定義

小さくて現在の出力解像度では読めない具体物(該当行・設定値・エラー文字)を、
話者が指示語で指している瞬間だけ大きく見せる。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 「このJSON」「ここの値」「このチェックボックス」「このエラー」など、
  画面上の具体物を指す語と、その対象名が同じ発話にある
- **補助(craft)**: 「これ」「ここ」だけでは対象を確定しない。前後の発話またはOCRで
  指示対象が一つに定まるときに限り使う

### 座標(frames --ocr の box / av の motion 領域)

- **決定的**: OCR box または知覚が与えた rect が存在し、その対象が全画面表示では
  小さく、文字や状態を判読するために局所拡大が必要
- **補助**: 実データの zoom 4件はすべて 960×540 の rect。これは接地値であり、
  recipe が座標を生成・変更する根拠にはしない

### 画面(frames / OCR テキスト)

- **決定的**: OCR テキストが発話中の具体語(設定名・値・エラー文)と一致する
- **補助(craft)**: 重要でも、全画面のまま読める大きさなら `already-legible` を優先する

### 時間・格子(アンカーの形)

- **補助**: 実データの単発 zoom は 4.28秒と11.39秒。発話が対象を説明する区間に
  アンカーが重なることを確認し、尺だけで分類しない

## 既定の型

`zoom`。LLM はアンカー番号と型だけを返し、rect・時刻・倍率は書かない。

## 反例(この型を当てない場合)

```text
「この仕組みが大事です」— 概念を指す「この」で、画面上の具体物が一つに定まらない。
→ concept-talk(none)。

「設定値は1280です」— 対応するOCR文字が既に大きく明瞭に読める。
→ already-legible(none)。
```

## 紛らわしい隣

- `already-legible` — OCR文字がある点は同じ。現在の解像度で読めない具体物を
  発話が指す場合だけ `tiny-target`。このG2対比は両 recipe から双方向に参照する
- `focus-shift` — 対象が一つの場所に留まるなら本分類、説明対象が画面内を移るなら
  `focus-shift`
