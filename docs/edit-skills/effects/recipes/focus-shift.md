# focus-shift

> 系: 見せる · 既定の型: zoom
> 接地: 実データ(2026-07-12, 連鎖1組) + rules.md + craft

## 一行定義

説明対象が画面内の別の場所へ移ったとき、全画面へ戻さず隣接 zoom を切り替えて
注視先を追う。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 「次にこちら」「今度は右側」「このエディターで」など、前の対象から
  別の具体物へ説明を移す発話がある
- **補助(craft)**: 接続語だけで決めず、前後アンカーの対象名またはOCRの違いを確認する

### 座標(frames --ocr の box / av の motion 領域)

- **決定的**: 隣接アンカーの rect 中心が異なり、同じ rect の継続では新しい対象を
  見せられない。座標自体は知覚の実在値をそのまま使う
- **補助**: 実データでは rect が `[406,426 960x540]` から
  `[857,0 960x540]` へ移動した

### 画面(frames / OCR テキスト)

- **決定的**: 前半と後半の代表 frame で、発話が指すUI要素が別位置にある
- **補助(craft)**: 単なるカメラ揺れや小さいレイアウト変化は対象移動とみなさない

### 時間・格子(アンカーの形)

- **決定的**: 前の zoom の `end` と次の zoom の `start` が一致する隣接連鎖で、
  説明も途切れず別対象へ移る
- **補助**: 実データの連鎖は `[433.29-445.32]` と `[445.32-454.80]`

## 既定の型

`zoom`。隣接アンカーごとに選び、LLM が中間座標や遷移時刻を作らない。

## 反例(この型を当てない場合)

```text
同じ設定値を言い換えながら説明しており、rect も同じ場所に留まる。
→ tiny-target の単一 zoom を継続し、focus-shift を増やさない。

画面がスクロールして注視先を自然に運んでいる。
→ motion-carries(none)。
```

## 紛らわしい隣

- `tiny-target` — 小さい対象を初めて見せる理由は `tiny-target`。その注視先が
  次の場所へ移る理由があるときだけ `focus-shift`
- `motion-carries` — 画面自身の遷移が注視先を明示するなら zoom の追従を足さない
