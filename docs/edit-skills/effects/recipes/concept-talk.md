# concept-talk

> 系: 何もしない · 既定の型: none
> 接地: 実データ(2026-07-12, zoom外区間) + rules.md + craft · 観測: none理由は未記録

## 一行定義

概念・導入・方針のトークだけで、画面上の具体物を指していないため演出を置かない。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 「考え方」「大事なこと」「〜を目指したい」など抽象的な方針を語り、
  画面上のファイル名・値・UI名への指示語がない
- **補助**: 「この」があっても「この考え方」のように概念を受けるなら具体物ではない

### 座標(frames --ocr の box / av の motion 領域)

- **決定的**: 発話に一対一で対応する局所 rect がない(`(領域なし)` を含む)
- **補助(craft)**: 偶然OCR boxがあっても、発話と結び付かない座標へ zoom しない

### 画面(frames / OCR テキスト)

- **補助**: タイトルや話者画面など、概念トークを支える全景はあるが、局所的に
  読ませる対象はない
- **補助(craft)**: 全景が意味を持つ場面は、局所演出より構図の安定を優先する

### 時間・格子(アンカーの形)

- **補助**: speech アンカーだけで座標が無い場合に多い。ただしspeech由来という
  だけで決めず、具体物への言及の有無を先に見る

## 既定の型

`none`。概念に対応する架空の座標を作らない。

## 反例(この型を当てない場合)

```text
「承認の仕組みが大事です。このハッシュが変わると失効します」
— 後半は画面上の具体的な値を指している。
→ tiny-target(zoom)を検討する。

概念説明中でも、画面にAPIキーが映り込んでいる。
→ 発話内容にかかわらず secret-exposure(blur)。
```

## 紛らわしい隣

- `already-legible` — 具体物は存在するが読めるので none なら
  `already-legible`。具体物への言及自体が無いなら `concept-talk`
- `motion-carries` — 画面の動きが注視を担うため none なら `motion-carries`。
  本分類は発話が抽象で局所対象がない
