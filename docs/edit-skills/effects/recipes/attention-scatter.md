# attention-scatter

> 系: 指す · 既定の型: annotation
> 接地: 想定(annotation機能仕様) + craft · 観測: 2026-07-12は0件

**このチャンネルの承認済み実データには annotation が無い。以下は機能仕様と
一般的な注視誘導から構成した接地なしの分類であり、worked example も合成例である。**

## 一行定義

説明対象が小さなwidget一つのzoom rectに収まらず、要素の多い画面で注視先が
散るため、囲み・矢印で対象を指す。

## 判定シグナル

### 語彙(transcript)

- **決定的**: 「左の一覧と右の結果」「上から2番目」「この流れ」など、全体の中の
  相対位置・関係を指す発話がある
- **補助(craft)**: 「ここを見て」だけでは不足。画面と合わせて対象が一意に定まる
  ときだけ使う

### 座標(frames --ocr の box / av の motion 領域)

- **決定的**: 対象の実在rectまたは点が知覚から得られ、周囲に似た要素が複数ある
- **補助(craft)**: 矢印・囲み・spotlightの形は座標から決まる幾何であり、この
  recipeの分類にはしない。LLMも形・座標を作らない

### 画面(frames / OCR テキスト)

- **決定的**: 同じ見た目の行・カード・ボタンが複数あり、発話だけでは視聴者が
  一目で対象を特定しにくい
- **補助(craft)**: 画面全体の関係を残す必要があるため、局所zoomでは文脈を失う

### 時間・格子(アンカーの形)

- **補助**: 対象を指す発話中に限る。発話後まで長く残す理由にしない
- **補助(craft)**: motionアンカーでは、動きが止まった後も対象が複数残るかを見る

## 既定の型

`annotation`。どのannotation形を使うかは知覚由来の幾何に任せる。

## 反例(この型を当てない場合)

```text
クリック対象が一つだけで、カーソル移動と選択状態がその場所を自然に示す。
→ motion-carries(none)。

対象は一つだが文字が小さく、周辺文脈を残す必要もない。
→ tiny-target(zoom)。
```

## 紛らわしい隣

- `motion-carries` — motionアンカーという表層は同じ。動きが対象を自明にするなら
  `motion-carries`、動き後も似た要素へ注意が散るなら `attention-scatter`。
  このG2対比は両recipeから双方向に参照する
- `tiny-target` — 判読のための拡大ならzoom、全体を保った注視誘導ならannotation
