# cutflow — AIネイティブ NLE 診断

*2026-07-06 / 「動画編集の Cursor」を狙う上での不足機能ギャップ分析*

> 調査方法: 7本の並列サブエージェント(内部コード監査4 + 外部/競合/AIネイティブ設計調査3)+ GUI 専用監査を統合。結論はソースコード実読と grep 検証に基づく。競合部は WebSearch + 知識カットオフ 2026-01。**リポジトリの「スコープ外/持たない機能」の記述はユーザー指示により無視し、すべて埋めるべきギャップとして評価した。** severity / effort / 合目は相対評価。

---

## 総合判定:Cursor で言えば「Composer / Agent 以前」

土台は正しい。**video-as-code(全編集が平文 JSON)+ `validate` / `describe` / `frames` の検証トライアド + ローカル所有 + 承認境界** は本物で、これはコードを AI ネイティブにしたのと同じ第一原理だ。しかし現状は「良い型とリンタを持つリポジトリに、**盲目で走る一発生成ボタン**が付いた」段階——2022〜23 の Copilot 期(賢い単発補完付きエディタ)相当で、その上に載るべき AI 協調レイヤは未着工。

Cursor の本質は二層構造:**(1) 人間が使う本物のエディタ(VS Code)** に **(2) AI 協調レイヤ(Composer / Agent / インライン diff 承認)** を載せ、**(3) データはユーザー所有・モデルは持ち込み**。cutflow をこの軸で採点する。

### スコアカード(何合目か)

| 軸 | 合目 | 一言 |
|---|---|---|
| 人間の手編集(GUI) | **7 / 10** | リアルタイム Player・波形・スナップ・undo/redo・直接ドラッグ。そこそこ本物 |
| 編集表現力(映像合成) | **3.5 / 10** | Premiere比3・ScreenStudio比4。キーフレーム基盤が無い |
| AI の知覚(目・耳) | **2 / 10** | 耳=文字起こしのみ / 目=proxy 静止画で画面文字が読めない |
| AI 協調レイヤ | **2 / 10** | 差分レビュー無し・GUI 内 AI 指示無し・MCP/編集 API 無し |

> 別軸:「**データ所有 × 持ち込み AI**」は思想としては本命の旗(◎)だが、実装は実質 Claude 専用(MCP / JSON Schema が無く、バックエンドは `claude-cli` / `api` の2種のみ)で、主張と実装が乖離している。

## 核心的発見:「目と耳を与えた」の一番肝心な工程が、盲目かつ聾

**カット判断 LLM が、映像も音声波形も見ていない。** 何を残す/切るか・章立て・ショート選定を決める `plan` は、`numberSegments` が作る「`#id [開始-終了秒] 発話text`」+ brief + 尺だけを LLM に渡している。`frames` の画像も、自前で計算した無音マップ(`cuts.auto.json`)すら入力に入らない。唯一の「目」である `frames` は**事後確認用の側チャネル**で、意思決定ループに接続されていない。編集品質は「文字起こしの良し悪し」に上限を縛られる。

> Cursor に喩えると → コードを実際に読むマルチモーダルなエージェントではなく、**要約テキストだけ読んで、ぼやけたスクショを片手にファイルを手書きするエージェント**。

これを裏付ける3つの事実:

- **目が、主役を読めない** — `frames` はベース映像を `proxy.mp4`(幅1280px)のアップスケールで描く。開発スクリーンキャストの主役=画面内のコード・ターミナル・エラーが判読不能。プロジェクト自身 `CLAUDE.md` が「可読性は判断できない」と認めている。
- **耳が、片方しか無い** — 文字起こしは `micWav` のみ。最終出力に mix されるシステム音声(デモ音・再生動画・TTS)は文字起こしされず AI から不可視。非言語音(笑い・間・強調・SFX)も話者分離も語単位タイムスタンプも取っていない。
- **承認ゲートが、紳士協定** — `validate` は `approved` を boolean 検査するだけ、`render` がそれをゲートする。つまり **AI が `approved:true` と書けば人間承認を丸ごとすり抜けて最終レンダーまで進める**。中核の安全設計がコードで強制されていない。

## 人間編集 × AI 編集(GUI)の現在地

**「AI に頼るだけでなくエディタでも編集できる」——ここが唯一登れている軸。** GUI は人間の手編集 NLE として**約7合目**:

- Remotion Player を `proxy.mp4` で走らせカット非焼込の即時プレビュー、マルチトラック、波形、スナップ/マグネット、ズーム可能タイムライン+仮想化、undo/redo(履歴100件)、プレイヤー上の直接ドラッグ配置(テロップ9点/素材8ハンドル/ズーム rect)、`⌘K` 分割、複数選択+一括スタイル、GUI からの preview/render 起動、未保存自動退避(`.editor-draft.json`)、保存前 `validate` ゲート。**「本物の NLE」水準に達している実装**。

しかし **その上の AI 協調レイヤは約2合目**——ここが本製品最大のギャップ:

| ギャップ | severity | effort |
|---|---|---|
| **AI 編集の差分レビュー / accept-reject が皆無**。外部/AI が JSON を書き換えても「未保存なし=黙って全再読込」or「未保存あり=読み直す(自分の編集破棄)/ ⌘S で相手を全上書き」の二値のみ。差分表示ゼロ・区間/フィールド単位マージなし。Cursor のインライン diff 承認に相当するものが無い | blocker | XL |
| **GUI から LLM コマンドを起動できない**(plan/remeta/plan-shorts/describe)。人間はターミナルへ降りる必要があり、GUI 内に AI 指示チャネルが一切無い | major | L |
| `colorFilter` / `thumbnail` / `meta` / `chapters` が GUI 非対応。AI は編集できるが人間は JSON 手編集に追い込まれる非対称 | major/minor | M |
| コピー&ペースト無し / リップル・ロールトリム無し / フレーム・タイムコード入力無し / マーカー・イン点無し | minor〜major | S〜L |

器(**正データ=ファイル + ホットリロード追従**)は協調 UI を後載せできる正しい土台。だが「AI と人間が同じタイムラインを協調で仕上げる」水準には構造的に未到達。

## 6テーマの不足(効き順)

### A. AI の知覚強化(目・耳)
「目耳を与えた」の中核だが、実装は文字起こしという片耳に痩せている。ここが埋まらない限り主張全体が崩れる。

- **plan に目耳が接続されていない** — `blocker` / effort L — frames 画像・無音マップ・音量特徴を意思決定に流し込む(or multimodal パス)。主張の生命線。
- **目が proxy 静止画で画面内テキストを読めない** — `blocker` / effort M — フル解像度 still 経路 + 画面領域 OCR。flagship 用途で最大の実用効果。
- **語/トークン単位タイムスタンプ・信頼度を使っていない** — `major` / effort **S（安い勝ち）** — `-ml/-owts` で語粒度へ。字幕タイミング・言い直しの語単位トリム・誤認識フラグが開く。
- **非言語音・システム音声・話者分離・keep 内の間が不可視** — `major` / effort M — 笑い・強調・SFX・デッドエアを音から拾えず、ショート/間詰め/対話編集ができない。
- **素材(B-roll)の中身を AI が知る手段が無い** — `major` / effort M — 得られるのはファイル名だけ。尺超過や不適切配置に気づけない。
- **`describe` が非可逆な散文要約(発話36字切り捨て)で機械可読な完全射影が無い** — `minor` / effort S。

### B. AI の行動インターフェース(MCP / 編集API / 差分・適用・undo)
行動手段は「生 JSON をファイルツールで手書き」の一択。編集プリミティブ・MCP・tool 定義が0で、不変条件は事後 `validate` に祈るしかない。

- **編集の action space が無い(MCP / 高水準プリミティブ)** — `blocker` / effort L — 「この区間をカット」「テロップ追加」を安全・発見可能な tool として露出。主張#1と#2を同時に前進。
- **安定 ID / @-mention が無い** — `major` / effort M — segment に永続 id が無く指示が全て曖昧アドレス。diff・差分適用・rules の共通基盤。
- **検査付きアトミック適用が AI から使えない** — `major` / effort M — GUI の `/api/save` は既に validate 前置の全拒否を実装済み。同じ経路を CLI/MCP へ露出するだけ。
- **undo / diff / checkpoint と承認スコープ強制が無い** — `major` / effort M — approved・中間生成物への書込を機械強制する安価な保険。
- **内部 LLM が1ショットで、エージェントループ(観測→再調整)になっていない** — `major` / effort L — `complete()` は `claude -p` を1回叩くだけ。予測的な次アクション(Tab 相当)も無い。

### C. 編集表現力(NLE パリティ)
カット主導の合成としては堅実だが、動きの語彙(キーフレーム/速度/トランジション)と画面内強調(注釈/ぼかし)が構造的に薄い。**天井を決める2つの設計判断**:(1) 時刻の「元収録秒 1:1 写像」→ 速度変更・クロスディゾルブが構造的に不可能。(2) canned アニメのみ → 任意プロパティのキーフレーム基盤が無い。

- **注釈グラフィックが皆無(矢印・囲み・スポットライト・図形)** — `blocker` / effort L — 描画プリミティブが無く、ズーム以外に「ここを見ろ」を示せない。dev screencast の核。
- **領域ぼかし/モザイクが無い(APIキー・PII の目隠し)** — `blocker` / effort M — 隠すにはカットか外部PNGのみ。事故リスク直結の割に実装は軽い=**安い勝ち**。
- **速度変更・フリーズフレームが無い** — `blocker`(dev用途) / effort XL — ビルド待ちの倍速=dev 最頻出要求にカット一択。「1:1 時刻写像」を壊すため後付け最重。
- **キーフレーム基盤が無い** — `major`(プロNLEには blocker) / effort XL — 動きは個別ハードコードの canned のみ。**多くの上位ギャップの共通の親**。
- **テロップ/タイトルのアニメーションが無い** — `major` / effort M — 素材にある `fadeIn/Out` すらテロップに無い非対称(埋めるのは安い)。
- カラーグレーディング(3スカラー全編一律)/ 音声深さ(区間音量・EQ・効果音レーン無し)/ 合成(クロマキー・マスク・トラッキング無し)/ マルチカム / オートリフレーム も未達。

### D. 高速フィードバックループ
`validate/describe/frames` は typecheck/test/lint の良い triad。だが AI が「編集を見る」ループは低帯域・低速。

- **フィードバックが動きなし・音なし・proxy 静止画** — `major` / effort L — 実 A/V は数分かかり AI は再生も試聴もできずループが人間で終端。テンポ・ダッキング・繋ぎを自己検証できない。
- **編集後の自動検証・視覚的アサーションが無い** — `minor` / effort M。
- **`frames` の stale-PNG 罠 + 毎回コールド再バンドル** — `minor` / effort S — 常駐 frames サーバが無く、微調整ループが毎回 Chrome 起動コストを払う。

### E. ローカル×持ち込み AI の運用基盤(ルール / 索引 / コンテキスト)
「任意 AI で編集」を支える基盤が薄い。契約は200行の日本語散文 `CLAUDE.md` に散在し、実質 Claude 前提。

- **機械可読な契約が無い(JSON Schema 0件)** — `major` / effort M — AGENTS_CONTRACT.md 型の能力/不変条件マニフェストを出せば非 Claude(codex 等)も守って編集できる。主張#2の要。
- **チャンネル/シリーズ単位の rules・学習が無い** — `major` / effort **S（安い勝ち）** — テロップ様式・声色・禁止語・ペーシングを rules 化。人間の上書きから学ぶ。`.cursor/rules` 相当。
- **意味索引/検索が無い(収録横断・長尺 retrieval)** — `major` / effort L — 全 transcript を1プロンプトに詰めるため60分収録で破綻。`@codebase` 相当の recall が要る。

### F. ポジショニング / 差別化
「チャットボット的接続からの脱却」を掲げるが、内部 AI 利用が1ショットの盲目バッチ=まさにそれを自分で体現している。

- **対話一発編集の製品化が無い(=最大の遅れ)** — `table stakes` / effort L — Descript Underlord / Adobe Prompt-to-Edit が「言えばやる」を標準化。プリミティブは揃うのに skill 化されていない。
- **BYO-AI が実質 Claude 専用** — `major` / effort M — バックエンド2種のみ・MCP 無し。B(MCP)/E(契約)と一体で解けば codex 等が一級市民に。
- **ローカルの普及税 / 完全ローカルではない点** — `major` / effort L — ツールチェーン自力構築が前提で実質エンジニア限定。加えて**発話テキストは外部 LLM へ渡る=「完全ローカル」ではない**(正直な訴求が必要)。

## 主張の検証:擁護し、突く

### Steelman — 方向は正しい
- 全編集を平文 JSON にしたのは、コードを AI ネイティブにしたのと同じ第一原理。ファイルを読み書きできるエージェントなら原理的に全成果物を著述できる。
- 時刻はツールが写像するので **LLM は秒の算術をせず番号選択だけ**——ハルシネーション耐性が構造的に組み込まれている。
- `validate / describe / frames` は typecheck/test/lint に相当する**本物の編集→検証ループ**。多くの競合の「タイムラインにチャットを貼っただけ」より先行している。
- **ローカル + ファイルが正データ**はベンダーロックを排し、既存 Claude 契約で API コスト0。閉じた AI 動画 SaaS より耐久性のあるアーキテクチャ。
- `approved` ゲートは「AI がドラフト、人間が出荷」という健全な分業を明示。

### 弱点 — 最重要地点での乖離
- **目が flagship 用途で事実上盲目**:frames は proxy 1280px アップスケール。開発デモの主役=コード/ターミナル/エラーが読めない。
- **肝心の plan が盲目かつ聾**:カット/章/タイトルを発話テキストだけで決め、フレームも音声波形も無音マップも入力しない。主張と実装が乖離。
- **行動層が生 JSON 手書きで安全網ゼロ**:`approved:true` を書けば人間承認をすり抜けて最終レンダーできる。中核の安全設計が紳士協定。
- **内部 AI が1ショットの盲目バッチ**:批判対象の「チャットボット的接続」を自分で体現し、差別化ナラティブが自己矛盾。
- **BYO-AI は実質 Claude 専用**:MCP も JSON Schema も無く、契約は日本語散文。**普及税**も高く、発話テキストは外部 LLM へ渡るので「完全ローカル」でもない。

## 競合地図:堀は「機能」ではなく「フォーマット」

市場はクラウド SaaS + 固定 AI + クレジット課金に収束。cutflow の旗は空白地帯——**機能で勝つ層ではなく、所有権・フォーマット・モデル選択の自由で勝つ層**。

| ツール | AI が担う編集範囲 | ローカル/クラウド | AI 固定/持込 | データ所有 |
|---|---|---|---|---|
| **cutflow** | 文字起こし編集・自動カット・ショート・テロップ | **ローカル** | **持込(自分の Claude)** | **ユーザー(平文JSON)** |
| Descript + Underlord | + 対話エージェント編集・一部生成 | クラウド | 半固定(ピッカー・クレジット) | クラウド専有文書 |
| Opus Clip | 自動ショート・テロップ・AI B-Roll | クラウド | 固定 | クラウド |
| Adobe Premiere (Firefly) | Text-Based Editing・Generative Extend・Prompt-to-Edit | ローカル+クラウド生成 | 固定 | ローカル/生成はクラウド |
| DaVinci Resolve 20 | IntelliCut・AI音声・多カメラ(補助止まり) | **ローカル** | 固定(内蔵) | **ローカル** |
| Screen Studio | 自動ズーム・カーソル演出(AI意味編集なし) | **ローカル(mac)** | ほぼなし | **ローカル** |
| TimeBolt / Recut | 無音・フィラー自動カットのみ | ローカルプラグイン | ルールベース | 母体NLE |
| CapCut (+AI) | 自動カット・ショート・テロップ・生成 | クラウド+アプリ | 固定 | クラウド(ByteDance) |

### 差別化の旗(立てられる所)
1. **video-as-code**〔差別化 強・本命〕— 全編集が平文 JSON。grep/diff/git/validate でき、任意のコーディングエージェントがファイルとして編集できる。他社は囲い込みが崩れるので踏み込めない。
2. **BYO-AI × 限界コスト0**〔強〕— 契約済み Claude をサブプロセス実行=動画1本あたり AI 課金ゼロ。他社は AI 推論コスト転嫁が収益源なので構造的に取れない。
3. **ローカル × データ主権**〔中〜強〕— 機密収録に効く。「ローカル × AI エージェント編集」の交点はほぼ空白。※発話テキストは外部 LLM へ渡る点は正直に。
4. **開発スクリーンキャスト特化**〔中〕— Screen Studio は美しい演出を握るが AI 意味カットを持たない。両取り。Cursor 型ビーチヘッド戦略。
5. **決定論・検証・オープン合成**〔中〜弱〕— validate 事前検証・差分レンダー・Remotion をコード拡張可能。

### 追いつくべき点(競合が持ち cutflow が弱い)
対話一発編集の製品化(**最優先**)、語単位フィラー除去、自動リフレーム、AI B-roll 自動挿入、AI 音声強調、アニメ字幕テンプレ。

### 生成 AI 統合の方針
生成 AI は**コア編集ループの外**に置き、`materials/` を埋める差し替え可能プラグイン(BYO-key)として扱う。コア(カット/テロップ/演出/合成)はローカル・決定論・データ主権のまま保つ。Runway/Sora/HeyGen/ElevenLabs/Suno を中核に据えると SaaS 化して旗を降ろすことになる。整合する統合点:生成映像→`materials/` に populate、生成 BGM→`materials/` + `bgm.json`、音声強調→ローカル ffmpeg/rnnoise 第一。アバター系は実写 dev 用途とずれるので最劣後。

## 結論:「AI ネイティブ NLE の Cursor」を名乗るための 5 条件

1. **tool 呼び出しを持つエージェントループ** — MCP による編集 action space、安定 ID、検査付きアトミック適用。`plan` を1ショット completion からエージェントループ(観測→再調整)へ。
2. **意思決定に効くマルチモーダル知覚** — 読める目(フル解像度 + OCR)、目耳を `plan` に接続、非言語の耳。「目耳を与えた」を約束から実装へ。
3. **インライン差分の accept/reject + checkpoint/undo** — GUI に AI 編集の区間/フィールド単位 diff レビュー。土台(ホットリロード・validate ゲート)は正しいので、上に協調 UI を載せる。
4. **安定アドレッシングと持続する rules/memory** — @-mention と、チャンネル単位の rules + 人間の修正から学ぶ層。動画間で「自分の色」を保つ。
5. **エージェントが消費できる速い実 A/V フィードバック** — 動きと音のある検証。加えて非専門家向けの「対話一発編集」を製品として skill 化——これが競合に対する最大の遅れ。

> 土台(データモデル + リンタ + ローカル所有 + 承認境界)は正しく置けている。この 5 点が揃って初めて、cutflow は「良い型を持つリポジトリに付いた一発生成ボタン」から「動画編集の Cursor」へ変わる。**堀は機能ではなくフォーマット**——JSON スキーマをオープンで安定した video-as-code 標準に育てきることが本業だ。
