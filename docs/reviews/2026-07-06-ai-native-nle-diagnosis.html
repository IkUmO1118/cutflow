cutflow / AIネイティブ NLE 診断 · 2026-07

「動画編集の Cursor」まで、
土台は正しい。上の階が空いている。

全編集を平文 JSON にした video-as-code の発想と、validate / describe / frames という検証トライアドは本物だ。だが「AI に目と耳を与えた」という主張の最重要地点が実装で成立していない——カット判断の LLM は映像も音声波形も見ず、文字起こしテキストだけで編集している。

総合判定
Cursor で言えば Composer / Agent 以前。2022–23 の Copilot 期(賢い単発補完付きエディタ)相当。良い型とリンタを持つリポジトリに、盲目で走る一発生成ボタンが付いた段階。
Cursor 型の二層構造で見ると
(1) 人間が使う本物の NLE = そこそこ本物。(2) その上に載る AI 協調レイヤ = ほぼ空。土台は正しく置けているが、エージェント体験は未着工。

核心的発見

「目と耳を与えた」の一番肝心な工程が、盲目かつ聾

カット判断 LLM が、映像も音声波形も見ていない

何を残す/切るか・章立て・ショート選定を決める plan は、numberSegments が作る「#id [開始-終了秒] 発話text」+ brief + 尺だけを LLM に渡している。frames の画像も、自前で計算した無音マップ(cuts.auto.json)すら入力に入らない。唯一の「目」である frames は事後確認用の側チャネルで、意思決定ループに接続されていない。編集品質は「文字起こしの良し悪し」に上限を縛られる。

Cursor に喩えると → コードを実際に読むマルチモーダルなエージェント ではなく、要約テキストだけ読んで、ぼやけたスクショを片手にファイルを手書きするエージェント。
目が、主役を読めない
frames はベース映像を proxy.mp4(幅1280px)のアップスケールで描く。開発スクリーンキャストの主役=画面内のコード・ターミナル・エラーが判読不能。プロジェクト自身 CLAUDE.md が「可読性は判断できない」と認めている。
耳が、片方しか無い
文字起こしはmicWavのみ。最終出力に mix されるシステム音声(デモ音・再生動画・TTS)は文字起こしされず AI から不可視。非言語音(笑い・間・強調・SFX)も話者分離も語単位タイムスタンプも取っていない。
承認ゲートが、紳士協定
validate は approved を boolean 検査するだけ、render がそれをゲートする。つまり AI が approved:true と書けば人間承認を丸ごとすり抜けて最終レンダーまで進める。中核の安全設計がコードで強制されていない。

A

AI の知覚強化(目・耳)

「目耳を与えた」の中核だが、実装は文字起こしという片耳に痩せている。ここが埋まらない限り主張全体が崩れる。

  • plan に目耳が接続されていないblockereffort Lframes画像・無音マップ・音量特徴を意思決定に流し込む(or multimodal パス)。主張の生命線。
  • 目が proxy 静止画で画面内テキストを読めないblockereffort Mフル解像度 still 経路 + 画面領域 OCR。flagship 用途で最大の実用効果。
  • 語/トークン単位タイムスタンプ・信頼度を使っていないmajoreffort S-ml/-owts で語粒度へ。字幕タイミング・言い直しの語単位トリム・誤認識フラグが開く。安い勝ち。
  • 非言語音・システム音声・話者分離・keep内の間が不可視majoreffort M笑い・強調・SFX・デッドエアを音から拾えず、ショート/間詰め/対話編集ができない。
B

AI の行動インターフェース

行動手段は「生 JSON をファイルツールで手書き」の一択。編集プリミティブ・MCP・tool 定義が0で、不変条件は事後 validate に祈るしかない。

  • 編集の action space が無い(MCP / 高水準プリミティブ)blockereffort L「この区間をカット」「テロップ追加」を安全・発見可能な tool として露出。#1と#2を同時に前進。
  • 安定ID / @-mention が無いmajoreffort Msegment に永続 id が無く指示が全て曖昧アドレス。diff・差分適用・rules の共通基盤。
  • 検査付きアトミック適用が AI から使えないmajoreffort MGUI の /api/save は既に validate 前置の全拒否を実装済み。同じ経路を CLI/MCP へ露出するだけ。
  • undo / diff / checkpoint と承認スコープ強制が無いmajoreffort Mapproved・中間生成物への書込を機械強制する安価な保険。多段編集の巻き戻しも。
C

編集表現力(NLE パリティ)

カット主導の合成としては堅実だが、動きの語彙(キーフレーム/速度/トランジション)と画面内強調(注釈/ぼかし)が構造的に薄い。

  • 注釈グラフィックが皆無(矢印・囲み・スポットライト)blockereffort L描画プリミティブが無く、ズーム以外に「ここを見ろ」を示せない。dev screencast の核。
  • 領域ぼかし/モザイクが無い(APIキー・PII の目隠し)blockereffort M隠すにはカットか外部PNGのみ。事故リスク直結の割に実装は軽い=安い勝ち。
  • 速度変更・フリーズフレームが無いblockereffort XLビルド待ちの倍速=dev最頻出要求にカット一択。「元収録秒 1:1 写像」を壊すため後付け最重。
  • キーフレーム基盤が無い / テロップアニメが無いmajor効き大動きは個別ハードコードの canned のみ。素材にある fade すらテロップに無い非対称(埋めるのは安い)。
D

高速フィードバックループ

validate/describe/frames は typecheck/test/lint の良い triad。だが AI が「編集を見る」ループは低帯域・低速。

  • フィードバックが動きなし・音なし・proxy静止画majoreffort L実 A/V は数分かかり AI は再生も試聴もできずループが人間で終端。テンポ・ダッキング・繋ぎを自己検証できない。
  • 編集後の自動検証・視覚的アサーションが無いminoreffort M「テロップが意図位置に載ったか」のプログラム的アサーションと render 回帰チェックが無い。
  • frames の stale-PNG 罠 + 毎回コールド再バンドルminoreffort S常駐 frames サーバが無く、微調整ループが毎回 Chrome 起動コストを払う。
E

ローカル×持ち込み AI の運用基盤

「任意 AI で編集」を支える基盤が薄い。契約は200行の日本語散文 CLAUDE.md に散在し、実質 Claude 前提。

  • 機械可読な契約が無い(JSON Schema 0件)majoreffort MAGENTS_CONTRACT.md 型の能力/不変条件マニフェストを出せば非 Claude(codex 等)も守って編集できる。主張#2の要。
  • チャンネル/シリーズ単位の rules・学習が無いmajoreffort Sテロップ様式・声色・禁止語・ペーシングを rules 化。人間の上書きから学ぶ。.cursor/rules 相当。安く効く。
  • 意味索引/検索が無い(収録横断・長尺retrieval)majoreffort L全 transcript を1プロンプトに詰めるため60分収録で破綻。@codebase 相当の recall が要る。
F

ポジショニング / 差別化

「チャットボット的接続からの脱却」を掲げるが、内部 AI 利用が1ショットの盲目バッチ=まさにそれを自分で体現している。

  • 対話一発編集の製品化が無い(=最大の遅れ)table stakeseffort LDescript Underlord / Adobe Prompt-to-Edit が「言えばやる」を標準化。プリミティブは揃うのに skill 化されていない。
  • BYO-AI が実質 Claude 専用majoreffort Mバックエンド2種のみ・MCP無し。B(MCP)/E(契約)と一体で解けば codex 等が一級市民に。
  • ローカルの普及税 / 完全ローカルではない点majoreffort Lツールチェーン自力構築が前提で実質エンジニア限定。加えて発話テキストは外部 LLM へ渡る=「完全ローカル」ではない(正直な訴求が必要)。

主張の検証

「AI に目耳を与えた NLE」を、擁護し、突く

Steelman — 方向は正しい

  • 全編集を平文 JSON にしたのは、コードを AI ネイティブにしたのと同じ第一原理。ファイルを読み書きできるエージェントなら原理的に全成果物を著述できる。
  • 時刻はツールが写像するのでLLM は秒の算術をせず番号選択だけ——ハルシネーション耐性が構造的に組み込まれている。
  • validate / describe / frames は typecheck/test/lint に相当する本物の編集→検証ループ。多くの競合の「タイムラインにチャットを貼っただけ」より先行している。
  • ローカル + ファイルが正データはベンダーロックを排し、既存 Claude 契約で API コスト0。閉じた AI 動画 SaaS より耐久性のあるアーキテクチャ。
  • approved ゲートは「AI がドラフト、人間が出荷」という健全な分業を明示。

弱点 — 最重要地点での乖離

  • 目が flagship 用途で事実上盲目: frames は proxy 1280px アップスケール。開発デモの主役=コード/ターミナル/エラーが読めない。
  • 肝心の plan が盲目かつ聾: カット/章/タイトルを発話テキストだけで決め、フレームも音声波形も無音マップも入力しない。主張と実装が乖離。
  • 行動層が生 JSON 手書きで安全網ゼロ: approved:true を書けば人間承認をすり抜けて最終レンダーできる。中核の安全設計が紳士協定。
  • 内部 AI が1ショットの盲目バッチ: 批判対象の「チャットボット的接続」を自分で体現し、差別化ナラティブが自己矛盾。
  • BYO-AI は実質 Claude 専用: MCP も JSON Schema も無く、契約は日本語散文。普及税も高く、発話テキストは外部 LLM へ渡るので「完全ローカル」でもない。

結論

「AI ネイティブ NLE の Cursor」を名乗るための 5 条件

tool 呼び出しを持つエージェントループ
MCP による編集 action space、安定 ID、検査付きアトミック適用。plan を1ショット completion からエージェントループ(観測→再調整)へ。
意思決定に効くマルチモーダル知覚
読める目(フル解像度 + OCR)、目耳を plan に接続、非言語の耳。「目耳を与えた」を約束から実装へ。
インライン差分の accept/reject + checkpoint/undo
GUI に AI 編集の区間/フィールド単位 diff レビュー。土台(ホットリロード・validate ゲート)は正しいので、上に協調 UI を載せる。
安定アドレッシングと持続する rules/memory
@-mention と、チャンネル単位の rules + 人間の修正から学ぶ層。動画間で「自分の色」を保つ。
エージェントが消費できる速い実 A/V フィードバック
動きと音のある検証。加えて非専門家向けの「対話一発編集」を製品として skill 化する——これが競合に対する最大の遅れ。

土台(データモデル + リンタ + ローカル所有 + 承認境界)は正しく置けている。この 5 点が揃って初めて、cutflow は「良い型を持つリポジトリに付いた一発生成ボタン」から「動画編集の Cursor」へ変わる。堀は機能ではなくフォーマット——JSON スキーマをオープンで安定した video-as-code 標準に育てきることが本業だ。

調査方法: 7本の並列サブエージェント(内部コード監査4 + 外部/競合/AIネイティブ設計調査3)+ GUI 専用監査 + 統合。ソースコード実読と grep 検証に基づく。競合部は WebSearch + 知識カットオフ 2026-01。リポジトリの「スコープ外」記述はユーザー指示により無視し、すべて埋めるべきギャップとして評価した。severity/effort・合目は相対評価。
