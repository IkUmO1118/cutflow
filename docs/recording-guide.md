# 収録ガイド(OBS セットアップと撮影チェックリスト)

cutflow は「拡張キャンバス方式」の収録を前提にする。
**キャンバス 3840x1080 の左半分に画面、右半分にカメラを並べて1本の mkv に録る。**
映像は1本でも空間的に分かれているので、パイプラインが後から画面とカメラを
別々に切り出せる(ワイプの位置・大きさは編集時に自由に決められる)。

> なぜこの方式か: OBS のマルチトラックは音声のみで、映像は必ず1本に合成される。
> 映像を分けて録るプラグイン(Source Record)は macOS で不安定なため、
> プラグイン不要で同期ズレが起きないこの方式を採用した(docs/decisions.md 参照)。

## OBS 初期設定(初回のみ、約30分)

### 1. 映像

設定 → 映像:

- 基本(キャンバス)解像度: **3840x1080**(手入力)
- 出力(スケーリング)解像度: **3840x1080**
- FPS: **30**

### 2. シーン構成

シーン「coding」:

- ソース「ウィンドウキャプチャ」(ターミナル/エディタ)
  → 位置 (0, 0)、サイズ 1920x1080 に変形(右クリック → 変換 → 変換の編集)
- ソース「映像キャプチャデバイス」(内蔵カメラ。MacBook の蓋を開けて使用)
  → 位置 (1920, 0)、サイズ 1920x1080

シーン「shiryo」(資料・ブラウザ用):

- coding と同じ構成で、ウィンドウキャプチャの対象だけブラウザ等に変える
- カメラは同じ配置のまま(シーン切替でカメラ位置が飛ばないように)

### 3. 音声(2トラック分離)

- 設定 → 音声 → マイク音声: **Osmo mini(USB 接続)** を選択
- システム音声が必要な場合はソース「macOS オーディオキャプチャ」を追加
- **編集 → オーディオの詳細プロパティ** で振り分ける:
  - マイク(Osmo): トラック **1** のみチェック
  - macOS オーディオキャプチャ: トラック **2** のみチェック

文字起こしはトラック1だけを使うため、キーボード音や通知音に汚されない。

### 4. 録画出力

設定 → 出力 → 出力モード「**詳細**」→ 録画タブ:

- 種別: 標準、録画フォーマット: **mkv**(Matroska)
- エンコーダ: **Apple VT H264 ハードウェアエンコーダ**
- ビットレート: 12000 Kbps 程度
- 音声トラック: **1 と 2 にチェック**
- 録画ファイルのパス: `~/Movies/cutflow`

### 5. macOS 権限

システム設定 → プライバシーとセキュリティで OBS に許可:
画面収録 / カメラ / マイク

## 撮影前チェックリスト(毎回)

- [ ] Osmo mini が USB 接続され、OBS の音声メーターが振れる
- [ ] MacBook の蓋が開いていてカメラに自分が映っている
- [ ] ターミナル/エディタのフォントは **20pt 以上**(初学者の見やすさ最優先)
- [ ] 録画対象ウィンドウをキャプチャ領域いっぱいに配置した
- [ ] macOS の集中モードをオン(通知が映り込まない)
- [ ] 画面に APIキー・個人情報が映るウィンドウ/タブがない
- [ ] **10秒テスト録画** → `node src/cli.ts ingest <フォルダ>` で
      「音声2トラック」と出ることを確認

## 撮影後

1. `~/Movies/cutflow/` に日付+内容のフォルダを作り、録画した mkv を入れる
   (例: `2026-07-03-whisper-bench/raw.mkv`)
2. `node src/cli.ts run <フォルダ>` を実行
