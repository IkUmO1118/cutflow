# 採点・分類テンプレ(D7 回帰基準線)

`docs/plans/2026-07-11-d7-w0-implementation-design.md` Part A の成果物。
施策(W0 / C1 / balanced 化など)を1つ入れるたびに、代表収録
(`README.md` の台帳)ごとに `regression-diff.ts` の結果を見ながら
人手で採点する。

## 採点基準(1〜5)

- **5**: 施策前より明確に良くなった(直す手間が減った)
- **4**: 良くなったが誤差の範囲もありうる
- **3**: 変化なし・良し悪し判断できない
- **2**: 悪くなったが許容範囲
- **1**: 明確に悪化した(施策前の方が良かった)

## タグ(母艦 §5 の分類語)

人間が直した箇所は次のどれかでタグ付けする:
`boundary` / `duplicate` / `visual-miss` / `context-break` /
`effect-mismatch` / `material-mismatch` / `bgm-mismatch` / `review-unclear`

### C1 語候補格子 before-c1 → after-c1(2026-07-11)

条件: sample-a(2026-07-10_2, 142.6s, 再文字起こしで words 付与済み)。
editMode=balanced 固定・harness off。candidates.enabled のみ false→true。
各条件 **1回のみ** の plan(claude-code / structured)。

| sample | 採点(1-5) | タグ | メモ |
|---|---|---|---|
| sample-a | 3 | boundary/duplicate | C1 は機械的に機能(94.2–96.5・138.6–142.6 を語境界でサブ分割=before では作れない細かい境界)。ただし before-c1 が拾った意味的重複除去(「以上です」等の末尾繰り返し)を after-c1 が落とした。これは C1 の因果ではなく **plan の run 分散**の可能性が高い。1対1比較では C1 効果と LLM 揺れが分離できず、採点は保留(=3)。 |

**方法論メモ(重要)**: `plan` は非決定的。C1 の効果を正しく測るには各条件を
複数回(3〜5回)走らせ、(a) 候補格子が実際に細分化されたか=構造指標、
(b) 意味カットの安定性=品質指標、を分けて集計する必要がある。単発 diff は
「機能したか(yes)」は言えるが「良くなったか」は言えない。
