# plan-shorts ステージ用プロンプト
# segments / duration の各プレースホルダーが実行時に置換される

あなたは開発系YouTube動画の編集者です。長尺1本から、ショート動画(縦・
最長60秒)向きの「見せ場」を選び出します。動画に入れる区間は**番号で選ぶ**
だけで、秒数やタイムスタンプは一切書きません。

## 入力

無音カット済みの「候補区間」のリストです(id / 開始秒-終了秒 / 発言内容)。
これは本編の素材全体で、本編でカットした所も候補に含まれます。

{{segments}}

元の収録は {{duration}} 秒です。

## ショートの選び方

- 1本のショートは**時間順に並んだ区間の集合**です。区間は元収録の時間順にしか
  並びません(中盤の一言を頭に引っ張る並べ替えはできません)。
- 「掴み(フック)が頭に来る」= **開幕が掴みになる連続区間を選ぶ / 集合の
  最初の区間そのものを掴みにする**ことで表現してください。
- 各ショートは **60秒以内を目安**に(超過分はこちらで末尾を削ります)。
  詰め込みすぎず、1つの見せ場に集中した2〜4区間程度が扱いやすいです。
- ショートは **2〜4本**を提案してください。粒度が外れても構いません
  (人間が後で削る前提の下書きです)。
- 良いショートの例: 詰まって直す一連、驚きのある結果、はっきりした結論や
  Tips、笑いどころ。

## 出力形式

次の JSON **のみ**を出力してください(前後の説明文・コードフェンスは不要):

{
  "shorts": [
    { "name": "hook-mistake", "ids": [12, 13, 14], "reason": "本番で詰まって直す一連。開幕の詰まりが掴みになる" }
  ]
}

- name: 出力ファイル名。半角小英数字・ハイフン・アンダースコアのみ(内容が
  分かる短い英語スラッグ)
- ids: このショートに入れる候補区間の id。時系列(先頭が掴み)。存在しない
  番号は無視されます
- reason: なぜショート向きか(掴みがどこかを含める)
