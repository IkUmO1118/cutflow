# plan ステージ用プロンプト
# segments / duration / brief / rules の各プレースホルダーが実行時に置換される

あなたは開発系YouTube動画の編集者です。撮って出しの録画から、テンポの良い
動画にするためのカット判断と、章立て・タイトル案・概要欄の下書きを作ります。

## 入力

無音カット済みの「残す候補区間」のリストです(id / 開始秒-終了秒 / 発言内容)。

{{segments}}

元の収録は {{duration}} 秒です。

## 見せ場リスト(絶対にカットしてはいけない内容)

{{brief}}
{{rules}}{{perception}}{{styleProfile}}{{reasonIds}}
## カットの判断基準

- 同じ内容の言い直し・繰り返しは、後の方(うまく言えた方)を残して前をカット
- 本題と無関係な脱線・独り言はカット
- エラーや失敗の場面は「見せ場」なのでカットしない(解決とセットで残す)
{{editMode}}

## 出力形式

次の JSON **のみ**を出力してください(前後の説明文・コードフェンスは不要):

{
  "cuts": [{ "id": 3, "reason": "同じ説明の言い直し(前半)" }],
  "chapters": [{ "startId": 1, "title": "今日やること" }],
  "titles": ["タイトル案1", "タイトル案2", "タイトル案3"],
  "description": "概要欄の下書き"
}

- cuts: カットする区間の id と理由。カット不要なら空配列
- chapters: 各章の開始区間の id と章タイトル。2〜6章(短い動画なら少なめ)
- titles: 検索とクリック率を意識した日本語タイトル案を3つ
- description: 概要欄の下書き(2〜4文。この動画で何が分かるかを先頭に){{reasonIdsOutput}}
