# JSの replace(文字列指定)は最初の1箇所しか置換しない+コメントにプレースホルダーを書くな

プロンプトテンプレートの置換で2つのミスが重なり、壊れたプロンプトが LLM に
送られた。

1. テンプレートの説明コメントに `{{segments}}` 等のプレースホルダー表記
   そのものを書いていた
2. `String.prototype.replace` を文字列指定で使っていた(最初の1箇所しか
   置換されない)

結果、コメント行のプレースホルダーが置換を消費し、本文は未置換のまま残った。
LLM は壊れた入力でも「それなりに」応答するため、普通のバグと違って
即エラーにならず気づきにくい。今回は `claude -p` が JSON の代わりに
バグ診断レポートを返してきたことで発覚した(生応答を毎回ファイルに
保存していたから調査できた)。

対策:

- テンプレートのコメントにはプレースホルダーの実表記を書かない
- `replaceAll("{{x}}", () => value)` を使う。関数形式にするのは、値に
  `$&` などの特殊置換パターンが含まれても解釈されないようにするため
- LLM の生応答は必ずファイルに残す(失敗モードが「エラー」ではなく
  「想定外の応答」になるため、後から見られることが唯一の手がかりになる)
